# Patreon content system

## Positioning

Core line: **Public channels show the output. Patreon shows the process.**

Patreon should feel early, useful, and honest. It is not a content dump. It is the place for drafts, decisions, previews, failed attempts, roadmaps, and supporter influence.

## Source material

- `changemaker`: daily shipped/learned/next notes.
- `subcult.tv`: longer ideas; create companion posts and public teasers.
- Public projects: milestones, screenshots, before/after, technical decisions, postmortems.

## Post lanes

1. Behind the changemaker: what changed, why, what broke, next.
2. Project field notes: goal, current state, obstacle, decision, next milestone.
3. Drafts/fragments/prototypes: unfinished but useful process material.
4. Blog companion: extra context for a `subcult.tv` post.
5. Roadmap / patron input: options, tradeoffs, vote/question.
6. Postmortem: goal, result, surprise, lesson, change.

## CTA stack

- Soft: “Follow along.”
- Medium: “Join Patreon for the full process.”
- Hard: “Support the work directly on Patreon.”

Rotate CTAs. Keep roughly one promo for every 3–4 value posts on public channels.

## Tone

Direct, makerly, low-hype, specific. Prefer concrete process nouns and verbs over abstract inspiration.

Avoid: “exclusive content,” hype claims, vague “community” language without a concrete action.
