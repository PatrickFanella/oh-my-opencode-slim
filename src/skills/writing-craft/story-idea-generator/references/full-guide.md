# Story Idea Generator — Full Guide

Detailed reference moved from `SKILL.md` for progressive disclosure.

## Core Principle

**Emotional experience first. Setting serves genre.**

“Sci-fi/fantasy/historical” are settings, not emotional genres.

## Elemental Genres (11)

- Wonder
- Idea
- Adventure
- Horror
- Mystery
- Thriller
- Humor
- Relationship
- Drama
- Issue
- Ensemble

Each genre is defined by reader emotional experience, not aesthetic wrapper.

## Genre Requirement Lens

For each primary genre, define:
- setting demands
- character demands
- plot demands
- thematic demand

Use this to reject setting details that do not serve emotional goal.

## Five-Phase Process

1. **Select emotional core**
   - choose primary genre
   - optionally add 1–2 secondary genres
2. **Choose/adapt setting module**
   - pick setting elements that serve genre
3. **Design characters for genre exposure**
   - vulnerabilities/strengths tied to core experience
4. **Develop high concept**
   - 1–2 sentence core pitch + central conflict arc
5. **Evaluate/refine**
   - genre clarity
   - setting-genre fit
   - character-genre fit
   - thematic resonance
   - originality

## Genre Combination Rules

- Secondary genre should enhance primary, not compete
- Keep secondary focus constrained (~30% rule)

Useful pairings include:
- Horror + Mystery
- Adventure + Wonder
- Thriller + Drama
- Mystery + Thriller

Problem pairings need control:
- Horror + heavy Humor
- Thriller + uncontrolled Relationship pacing

## Common Mistakes

1. Setting mistaken for genre
2. Secondary genre takeover
3. Checklist compliance without emotional result
4. Character-genre mismatch
5. Concept without genre foundation

## Diagnostic Sequence

1. Ask: “What should reader feel?”
2. Map concept to primary genre
3. Validate setting/character/plot alignment
4. Score and patch weak dimensions

## Example Pattern

If writer says “I want sci-fi,” force emotional clarity:
- wonder? thriller? horror? idea?

Then select setting expressions that reinforce that choice.

## Output Persistence

Persist:
- primary/secondary genre decision
- generated concept set
- character-positioning notes
- refined pitches

Suggested file pattern:
- `{concept-name}-{date}.md`

## Operating Boundaries

Do not:
- write full story for user
- force genre they do not want
- prioritize framework over inspiration
