# Statistical Distance — Full Guide

Detailed reference moved from `SKILL.md` for progressive disclosure.

## Core Principle

First instinct usually detects correct emotional need, but picks statistical-center expression (common cliché). Keep vector, increase distance.

Target: 60–80% familiarity (recognizable + fresh), not pure novelty.

## Center vs Edge

### Statistical Center
- immediate recognition
- reliable but overused
- low specificity, predictable path

### Statistical Edge
- same function, less crowded expression
- requires slight explanation
- creates narrative fertility

## Vector/Distance Method

1. Identify instinctive element
2. Name functional/emotional core
3. List center options (3–5 common defaults)
4. Push along same vector using transformation moves
5. Evaluate fertility + emotional-core retention

## Functional Core Reference

Common needs:
- stakes
- pressure
- sympathy
- complexity
- expertise
- access
- urgency
- moral weight

## Push Techniques

### A) Adjacent Substitution
Swap role/container to less common adjacent variant.

### B) Complication Layering
Add dimension that opens branches, not noise.

### C) Ironic Inversion
Flip expected relationship/dynamic.

### D) Category Jumping
Same emotional pressure, different domain.

### E) Specificity Injection
Concrete peculiar detail that forces downstream implications.

## Fertility Test

High fertility if edge choice:
- creates multiple subplot paths
- intersects relationships/conflicts
- raises productive questions
- evolves over story

Low fertility if it:
- solves one scene only
- sits inert after setup
- has single obvious resolution

## Genre Snapshots

### Noir
- Center: alcoholic ex-cop
- Edge: failed authenticator / documentary subject

### Romance
- Center: class difference + misunderstanding
- Edge: schedule-misalignment systems, asymmetric constraints

### Horror
- Center: isolated cabin, dead phone/car
- Edge: crowded-but-linguistically isolated, unusual embodied constraints

## Diagnostic Questions

### Detecting Center
- interchangeable with many stories?
- zero explanation needed?
- seen 10+ times?

### Confirming Edge
- still delivers original emotional core?
- creates extra story options?
- specific to this story world?

## Pitfalls

- pushing too far (lose core)
- complexity without purpose
- pushing everything to edge (reader fatigue)
- superficial substitutions with same dynamic
- edge elements with no fertility

## Workspace Method

Use three-column table:
- Functional need
- Statistical center options
- Statistical edge options

## Advanced Moves

- **Fertility chain**: edge choices that interlock
- **Specificity cascade**: one detail forces coherent implications

## Output Persistence

Persist:
- functional need statement
- center options
- edge options + rationale
- chosen edge + fertility notes

Suggested file pattern:
- `{story-name}-statistical-{date}.md`

## Verification Boundaries

High confidence:
- functional core clarity
- center enumeration completeness

Medium confidence:
- edge fertility estimation

Human judgment required:
- freshness vs forced
- distance calibration
- genre appropriateness

## Anti-Pattern Notes (Expanded)

2. Emotional-core loss
3. Complexity confusion
4. Adjacent-but-same dynamics
5. Fertility neglect
