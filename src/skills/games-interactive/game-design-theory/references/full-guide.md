# Game Design Theory — Full Guide

Detailed material moved out of `SKILL.md` for progressive disclosure.

## MDA Framework

- **Mechanics**: rules, actions, constraints, state changes.
- **Dynamics**: emergent behavior from mechanic interactions.
- **Aesthetics**: player experience/emotions (fun, tension, achievement, flow).

Design direction: Mechanics -> Dynamics -> Aesthetics.
Validation direction: Player feeling -> infer dynamic and mechanic causes.

## Core Engagement Loop

1. Input
2. Process
3. Feedback
4. Reward
5. Repeat

Quality targets:

- fast feedback (<100ms where interaction-critical)
- clear causation
- rewarding outcomes
- compelling repetition

## Flow Channel

Target difficulty to track skill growth. Too easy -> boredom. Too hard -> anxiety/frustration.

## Player Psychology

### Bartle types

- Achiever: goals/progression
- Explorer: discovery/secrets
- Socializer: community/co-op
- Killer: competition/PvP

### Self-Determination Theory

- Autonomy (choice/control)
- Competence (mastery)
- Relatedness (connection)

## Reward Systems

### Reward type

- Intrinsic: mastery, curiosity, expression, accomplishment
- Extrinsic: points, unlocks, currency, leaderboard rank

### Schedules

- Fixed ratio
- Variable ratio (use ethically)
- Fixed interval
- Milestone gates

## Balance Principles

| Aspect | Goal | Techniques |
|---|---|---|
| Mechanical | options all viable | counter-play, trade-offs |
| Economic | meaningful scarcity | sinks/faucets |
| Difficulty | challenge fit | dynamic scaling, curve smoothing |
| Competitive | fairness | mirror balance, anti-dominance tuning |

## Troubleshooting Patterns

### Bored players

Likely causes:

- challenge below flow band
- weak goals/progression
- slow feedback cycle

Fix:

- lift challenge ramp
- add milestones/rewards
- tighten loop + add variety

### Frustrated players / churn

Likely causes:

- difficulty spikes
- unclear mechanics/feedback
- perceived unfair randomness

Fix:

- smooth ramp
- improve onboarding/tutorialization
- make failure legible and educational

### Dominant strategy

Fix:

- add counters
- buff underused options
- introduce situational strength

## Design Checklist

### Pre-production

- audience defined
- core loop documented
- differentiator clear
- reference games analyzed

### Production

- mechanics serve target aesthetics
- feedback loops tested
- balance model maintained
- structured playtest cadence

### Polish

- first-time UX validated
- difficulty curve validated
- reward cadence tuned
- edge-case handling complete
