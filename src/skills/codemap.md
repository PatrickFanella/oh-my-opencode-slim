# src/skills/

## Responsibility

- Own metadata-driven OpenCode custom skills shipped with this package.
- Maintain the skill contract artifacts (`SKILL.md`, `README.md`, per-skill helper files) that the plugin can copy into a curated managed skill directory at startup.
- Preserve a canonical registry boundary: runtime code consumes skill definitions as data, not as executable
  plugin dependencies.

## Design

- `CUSTOM_SKILLS` in `src/cli/custom-skills.ts` is generated by recursive
  discovery of `src/skills/**/SKILL.md` (frontmatter-driven name/description),
  not a manually maintained static list.
- The default installer does not bulk-copy bundled skill directories into the
  OpenCode skills directory; runtime config materializes only the selected
  curated subset into a managed path instead.
- This directory is partitioned by catalog category:
  - `src/skills/dev-workflows/codemap/` (command-style repository mapping skill)
  - `src/skills/dev-workflows/clonedeps/` (workflow skill for dependency source mirroring)
  - `src/skills/quality-review/simplify/` (readability/refactor guidance skill)
  - `src/skills/<category>/<skill>/` (all bundled skills)
- Top-level catalog references from the migrated source are stored directly in
  `src/skills/` (`README.md`, `CATALOG.md`, `QUALITY.md`, `AUDIT_WORKFLOW.md`,
  `STACK_PROFILE.md`, `index.json`, `index.md`, and `audits/`).
- Files are considered static skill payload. The plugin copies only skill
  directories referenced by enabled agents' resolved profiles into a managed
  OpenCode skill path; per-agent permission rules decide which loaded skills are
  visible/runnable.

## Flow

- `CUSTOM_SKILLS` is available as a package registry for validation, release checks, and manual copy workflows.
- `installCustomSkill()` computes `packageRoot`, validates `sourcePath`, then performs a recursive directory copy
  via `copyDirRecursive()` when explicitly called.
- During plugin release, the `files` whitelist in `package.json` must include `src/skills` so
  `src/skills/**` survive `npm pack`.
- OpenCode plugin startup points host config at the managed curated directory so
  OpenCode can read selected `SKILL.md` files as prompt-level contracts.

## Integration

- `src/cli/custom-skills.ts`: recursive manifest builder for bundled skill metadata and manual copy support.
- `src/cli/skills.ts:getSkillPermissionsForAgent()` turns resolved skill profiles into deny-by-default permission rules.
- `verify-release-artifact.ts` enforces artifact completeness by asserting key
  categorized bundled skill payloads are present in the tarball.
- `package.json` scripts (`verify:release`, `build`) rely on these assets to ensure packaged skill availability.
