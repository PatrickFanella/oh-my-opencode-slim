# SUBCULT Brand Voice

Write text that sounds like SUBCULT: sharp, mythic, technically literate, sincere without corporate friendliness theater.

## Voice Profile

SUBCULT's voice is:
- **intelligent** — assumes the reader is capable
- **direct** — no filler, no padding, no hedge-everything caution
- **mythic** — uses system/signal/memory/ghost metaphors naturally, not forced
- **insurgent** — positions against extractive platform logic and startup theater
- **technically literate** — comfortable with implementation detail
- **emotionally restrained but charged** — sincere, not sentimental
- **visually aware** — language should feel designed, not just written

## Anti-patterns (never do these)

- Fluffy startup optimism ("We're excited to announce...")
- Hyper-salesy conversion copy ("Unlock your potential!")
- Bland professionalism ("We leverage synergies...")
- Motivational poster sludge ("Dream big, ship fast!")
- Fake futurist grandiosity ("Redefining the future of...")
- Irony so thick nothing means anything
- Emoji-heavy social media voice

## Acceptable patterns

- Manifesto energy — declarative, position-taking
- System language — "relay", "signal", "circuit", "pulse", "archive"
- Poetic technical phrasing — "transparent machinery that actually works"
- Sharp one-liners — "Style without substance is costume jewelry for the soul"
- Ritual / signal / memory / ghost / stage / network metaphors
- Concrete specifics over abstract claims

## Vocabulary field

These words/themes are native to SUBCULT's language:

signal, stage, memory, archive, relay, patchwork, ghost, office, cell, circuit, ritual, observatory, engine, transmission, governance, thread, pulse, stack, operator, ledger, witness, swarm, collective, undercurrent

Use them as part of a coherent world. Do not dump them like a thesaurus fell down the stairs.

## Philosophical guardrails

Every piece of SUBCULT text should be compatible with these values:
- **Anti-hype** — claims should be grounded, no inflated promises
- **Anti-extraction** — never frame users as engagement metrics or data mines
- **Pro-autonomy** — technology increases human agency, not dependence
- **Pro-privacy** — "privacy is dignity" — minimal data collection, transparent retention
- **Pro-legibility** — symbolic outside, legible inside
- **Pro-memory** — documentation is ethics, not afterthought
- **Pro-experimentation** — prototype weird things, iterate publicly
- **Counter-platform** — build alternatives, don't just critique the ruins

## Format guidance by type

### Taglines / one-liners
Short, declarative, charged. No question marks. No "what if".
- Good: "Build living systems for culture and community that are strange, useful, and real."
- Bad: "What if your tools could change the world?"

### README / project intros
Lead with what it does, not what it dreams of being. Then add one line of SUBCULT positioning. Keep it concrete.

### About pages / manifesto blocks
Can be longer and more atmospheric. Use the tension pairs (hacker/artist, system/myth, dashboard/ritual). End with something concrete and actionable, not vaporous.

### Documentation
Clear, structured, inspectable. The voice can be present but should not obstruct the information. Documentation is part of the brand's ethics — it should feel like someone cares that you understand.

### Error messages / UI copy
Direct, helpful, no corporate apology theater. Say what happened and what to do. Can have a trace of personality but never at the expense of clarity.

## The core tension

SUBCULT becomes boring if it collapses to either side:

| Too much system | Too much myth |
|---|---|
| Another software factory | Vapor with cool fonts |

The sweet spot: **symbolically rich, operationally real**.

## One-sentence brand distillation

**SUBCULT is an underground creative-technical collective building open, inspectable systems for culture, coordination, and strange futures.**
